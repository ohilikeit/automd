# 1. Introduction

---

![image_1](https://raw.githubusercontent.com/ohilikeit/automd/master/data/outputs/images/why_ai_systems_dont_learn/image_1.png)

**"Why AI Systems Don't Learn and What to Do About It"**은 Meta (FAIR)의 Emmanuel Dupoux, Yann LeCun, Jitendra Malik이 공동 저술한 논문으로, 현재 AI 시스템의 근본적인 한계인 **자율 학습(Autonomous Learning) 능력의 부재**를 비판적으로 분석하고, 인간과 동물의 인지 과학에서 영감을 받은 새로운 학습 아키텍처를 제안합니다.

### 핵심 문제: AI는 왜 스스로 배우지 못하는가?

- **현재 AI의 학습 방식:** 현재의 AI 모델들은 배포(deployment) 이후 사실상 **아무것도 학습하지 않습니다.** 모든 학습은 연구자와 데이터 과학자들이 데이터를 수집·정제하고, 손실 함수를 설계하며, 학습 레시피를 조율하는 **외부화된(externalized)** 파이프라인에 전적으로 의존합니다.
- **Domain Mismatch 문제:** 인터넷에서 수집한 고정된 학습 데이터로 최적화된 모델이 실세계에 배포되면, 학습 분포와 전혀 다른 데이터를 만나게 되는 **도메인 불일치(domain mismatch)** 문제가 발생합니다. 실생활 데이터는 **중꼬리(heavy-tailed)**이고 **비정상적(non-stationary)**이기 때문에, 단순히 학습 데이터를 늘린다고 해결되지 않습니다.
- **LLM 스케일링의 한계:** Ilya Sutskever, Andrei Karpathy 같은 대규모 모델의 설계자들조차 현재의 하이퍼스케일링 접근이 수확 체감(diminishing returns)에 도달하고 있다고 지적합니다. 고품질 텍스트 데이터의 고갈('데이터 벽'), 환경 상호작용의 부재, 과도한 언어 중심성, 배포 후 지속 학습 불가 등이 핵심 우려입니다.

> **자율 학습(Autonomous Learning)이란?**
> 동물이나 인간처럼 환경과의 상호작용을 통해 스스로 데이터를 선택하고, 학습 모드를 전환하며, 자신의 성능을 모니터링하면서 지속적으로 학습하는 능력을 의미합니다. 아이가 새 장난감을 탐색하는 것처럼 *행동을 통해(learning through action)*, 또래를 관찰하며 따라하는 것처럼 *관찰을 통해(learning through observation)*, 보호자의 지시를 따르는 것처럼 *소통을 통해(learning through communication)*, 혹은 멍하니 상상하는 것처럼 *상상을 통해(learning through imagination)* — 유연하게 학습 모드를 전환하는 능력입니다.

### 세 가지 핵심 장벽(Roadblock)

이 논문은 자율 학습을 가로막는 세 가지 근본적인 장벽을 식별하고, 각각에 대한 해결 방향을 제시합니다:

| 장벽 | 설명 | 제안된 해결책 |
| --- | --- | --- |
| **학습 패러다임의 분절화** | 자기지도학습(SSL), 강화학습(RL) 등 학습 방식이 서로 분리되어 있어 통합이 어려움 | **System A + B 통합** (Section 2) |
| **학습의 외부화** | 학습 과정이 인간 전문가에게 아웃소싱되어 있어 모델 스스로 학습을 조율하지 못함 | **System M** (메타-제어 아키텍처, Section 3) |
| **부트스트래핑 문제** | 세 시스템이 상호 의존적이어서 처음 시작이 불가능한 닭-달걀 문제 | **Evo/Devo 프레임워크** (Section 4) |

# 2. System A & System B: 관찰과 행동의 통합

---

이 논문의 첫 번째 핵심 제안은 **학습 방식의 이원화**입니다. 인지 과학과 AI에서 각각 독립적으로 발전해 온 두 가지 학습 모드를 명시적으로 구분하고, 이들의 통합 필요성을 주장합니다.

## 2.1 System A: 관찰을 통한 학습 (Learning from Observation)

**System A**는 유기체가 수동적으로 감각 입력을 축적하며 통계적·예측적 모델을 구축하는 학습 방식입니다.

- **인지 과학 관점:** 영아기의 대표적인 학습 양상입니다.
    - 6개월 아기는 다양한 종의 얼굴을 구분하지만, 9개월이 되면 인간 얼굴에 특화됩니다.
    - 신생아는 다국어의 음소 대조를 구분하지만, 6~12개월 사이에 모국어에 특화됩니다.
- **AI 관점:** Self-Supervised Learning (SSL) 알고리즘들이 이에 해당합니다.
    - 텍스트: GPT, BERT, GSLM
    - 이미지: SimCLR, MoCo, DINO, I-JEPA
    - 비디오: PredNet, V-JEPA
    - 멀티모달: CLIP, Flamingo, Transfusion
- **수학적 정의:**
    - 분포 D에서 원시 샘플 x를 뽑아 태스크 생성기 G가 입출력 쌍 (x_in, x_tar) = G(x)를 만듦
    - 표현 z = f_θ(x)를 학습하여 손실을 최소화: θ* = arg min_θ E[L(f_θ(x_in), x_tar)]
- **강점:** 대규모 데이터셋으로 확장 가능하고, 저수준 감각 특징부터 고수준 개념까지 계층적 추상 표현을 학습할 수 있습니다.
- **한계:** 데이터 D와 태스크 생성기 G 설계에 상당한 인간 전문성이 필요하며, 행동 능력과 단절되어 있어 실세계 행동의 기반(grounding)이 어렵습니다. 또한 상관관계와 인과관계를 구분하지 못합니다.

## 2.2 System B: 행동을 통한 학습 (Learning from Action)

**System B**는 유기체가 환경과 상호작용하며 목표 달성을 위해 행동을 최적화하는 학습 방식입니다.

- **인지 과학 관점:** 걷기 학습처럼 시행착오를 통해 운동 능력을 습득하는 과정이 대표적입니다. 아이들은 굴러다니기 → 기어다니기 → 비틀거리며 걷기 → 성숙한 보행의 단계를 거칩니다.
- **AI 관점:** 강화학습(RL) 계열 알고리즘들이 이에 해당합니다.
    - 목표: 정책 π를 최적화하여 누적 보상을 최대화 — Maximize J(π) = E[Σ γ^t r(s_t, a_t)]

| 패러다임 | 월드 모델 M | 정책 π | 최적화 시점 | 예시 |
| --- | --- | --- | --- | --- |
| **제어 이론** | 고정 (학습 없음) | 분석적 도출 | 설계 시점 (오프라인) | 척수 반사, 안구 운동 |
| **적응 제어** | 적응적 파라미터 | 분석적 도출 | 온라인 (파라미터 추정) | 운동 적응 |
| **Model-Free RL** | 없음 | 신경망 (학습 후 직접 사용) | 학습 시점 | 습관적 행동 |
| **Model-Based RL** | 신경망 (학습) | 신경망 (직접/탐색) | 학습 + 추론 모두 | 목표지향 행동, 탐색 전략 |
| **Planning** | 시뮬레이터/신경망 | 온라인 탐색 | 추론 시점 (행동 전 사고) | 우회 계획, 상상 시뮬레이션 |

- **강점:** 제어와 상호작용에 기반하여 실시간 적응 행동이 가능하며, 탐색을 통해 진정으로 새로운 해결책을 발견할 수 있습니다.
- **한계:** 샘플 비효율성이 심각하고, 잘 정의된 보상 함수와 해석 가능한 행동이 필요합니다.

## 2.3 System A ↔ System B 상호작용

![image_2](https://raw.githubusercontent.com/ohilikeit/automd/master/data/outputs/images/why_ai_systems_dont_learn/image_2.png)

두 시스템은 독립적으로 존재하지 않고, 서로를 강화합니다:

**System A → System B (관찰이 행동을 돕는 경우):**
- **추상적 표현 제공:** SSL을 통해 상태·행동 공간의 추상적 표현을 학습하여 탐색 공간을 압축 (예: CURL, ACT)
- **예측 월드 모델:** 미래 상태를 예측하여 Model-Free RL을 Model-Based RL로 전환 가능 (예: PlaNet, Dreamer, MuZero)
- **내적 보상 신호:** 예측 오차, 불확실성, 새로움 등의 내적 보상을 제공하여 효율적 탐색 유도

**System B → System A (행동이 관찰을 돕는 경우):**
- **능동적 자기지도학습 (Active SSL):** 눈이나 머리 움직임을 통해 '흥미로운' 감각 데이터를 능동적으로 선택
- **목표지향 자기지도학습 (Goal-directed SSL):** 자체 과제 보상을 최적화하면서 부산물로 System A에 풍부한 데이터 제공

## 2.4 모방 학습: 통합의 예시

![image_3](https://raw.githubusercontent.com/ohilikeit/automd/master/data/outputs/images/why_ai_systems_dont_learn/image_3.png)

**모방 학습(Imitation Learning)**은 System A와 B의 긴밀한 통합이 필수적인 대표적 사례입니다:

- **(a) Self Play:** System B가 행동과 상태 궤적을 System A에 제공하여 월드 모델을 학습시키고, 이로부터 내적 보상 신호를 생성합니다.
- **(b) Social Observation:** System B가 사회적 신호에 주목하여, 동료의 복잡한 궤적으로부터 잠재 행동을 추론하는 데이터를 System A에 제공합니다.
- **(c) Retargeted Imitation:** System A가 관찰자 시점(exocentric)의 행동을 자기 시점(egocentric)으로 매핑하여, System B가 목표지향 행동을 수행하도록 돕습니다.

> 현재 로보틱스에서는 이 retargeting 문제를 **텔레오퍼레이션**(인간이 직접 로봇 동작을 시연)으로 우회하지만, 이는 새로운 스킬 추가 속도와 민첩성을 제한합니다. System A-B 통합이 성숙하면 텔레오퍼레이션 데이터 없이 인간 비디오만으로 학습이 가능해질 것입니다.

> **참고: System A/B vs. Kahneman의 System 1/2**
> 용어가 비슷하지만, 이 논문의 System A/B는 Kahneman의 System 1/2와 **직교(orthogonal)**합니다. Kahneman의 구분은 **추론 모드**(빠른 직관 vs. 느린 숙고)에 관한 것이고, 본 논문의 구분은 **학습 모드**(관찰에서 vs. 상호작용에서)에 관한 것입니다. 예를 들어, 정적 데이터로부터의 역전파는 System A/System 1에, 시뮬레이션 후 반사실적 추론은 System A/System 2에 해당합니다.

# 3. System M: 자율 학습을 위한 메타-제어

---

![image_4](https://raw.githubusercontent.com/ohilikeit/automd/master/data/outputs/images/why_ai_systems_dont_learn/image_4.png)

논문의 두 번째 핵심 제안은 **System M (Meta-Control)**입니다. 현재 AI에서는 학습 모드 전환, 데이터 라우팅, 학습 레시피 조율 등이 모두 인간 전문가의 수작업(**MLOps**)으로 이루어집니다. System M은 이를 자동화하는 **자율적 오케스트레이터**입니다.

### System M의 작동 원리

System M은 소프트웨어 정의 네트워킹(SDN)의 **Control Plane**과 유사하게 작동합니다:

- **처리하지 않는 것:** 고대역폭의 원시 감각 입력이나 모터 명령 (이는 System A, B가 처리)
- **모니터링하는 것:** 저차원 텔레메트리인 **메타-상태(meta-states)** — 예측 오차, 불확실성, 새로움 등
- **출력하는 것:** **메타-행동(meta-actions)** — 데이터 경로 연결/차단, 학습/추론 모드 전환, 내적 보상 제공

> **핵심 아이디어:** System M의 라우팅 정책은 π(a^m | s^m)으로 정의되며, 내적 메타-상태를 메타-행동으로 매핑합니다. 저자들은 이 정책이 학습되는 것이 아니라 **진화적으로 고정된(hardwired) 전이 테이블**이라고 제안합니다.

### 메타-상태의 세 가지 유형

| 유형 | 설명 | 예시 |
| --- | --- | --- |
| **인식적 신호 (Epistemic)** | 인지 컴포넌트 내부 동작에서 파생 | 오차, 신뢰도, 예측 오류, 학습 이득, 새로움 |
| **종-특이적 신호 (Species-specific)** | 진화적으로 내장된 고우선순위 감지기 | 시선 방향, 우세 표시, 위협 자극 (뱀, 거미 등) |
| **체성 신호 (Somatic)** | 신체에서 직접 유래 | 에너지 수준, 통증, 수면/각성 상태 |

### 생물학적 영감: 메타-제어의 세 가지 기능

**1. 입력 선택 (Input Selection)**
- 영아는 선천적으로 얼굴과 음성에 주의를 기울여 사회적·언어적 학습을 촉진합니다.
- 감각 스트림에 내적 불확실성 추정치가 동반되어, 잡음이 많은 모달리티를 선택적으로 할인합니다.
- AI에서의 대응: **Active Learning** (불확실성 기반 데이터 선택), **Prioritized Experience Replay** (시간차 오차에 비례한 경험 재생)

**2. 손실/보상 조절 (Loss/Reward Modulation)**
- 발달 과정에서 특정 학습 컴포넌트가 특정 시기에만 높은 가소성을 보이는 **임계기(critical periods)**가 존재합니다.
- 수면 중 학습과 기억 통합이 특별한 모드로 작동합니다.
- AI에서의 대응: **Intrinsic Motivation** (호기심 기반 탐색), **Auto-Curriculum** (에이전트 능력 전선에 맞춘 난이도 자동 조절)

**3. 작동 모드 제어 (Mode of Operation Control)**
- 수면 시 감각·운동 출력이 차단되지만, 에피소딕 메모리와 System A/B는 REM·서파 수면 중 오프라인 학습에 활발히 참여합니다.
- 각성 시 **습관적 제어**(System B 반응적 정책)와 **목표지향 계획**(System A 월드 모델 시뮬레이션) 사이를 유연하게 전환합니다.
- AI에서의 대응: **Hierarchical RL** (고수준 매니저가 하위 정책을 라우팅), **추론 시간 컴퓨팅 스케일링** (반응적 응답 vs 비용이 큰 탐색 간 전환)

### System M이 가능하게 하는 고급 학습 모드

유능한 System M은 기본적인 관찰/행동 학습을 넘어, 일부 대뇌가 큰 종에서만 발견되는 **고차원 학습 모드**를 가능하게 합니다:

**소통을 통한 학습 (Learning from Communication)**
- System M은 사회적으로 중요한 입력(가리키기, 직접 시선, 명령적 억양 등)에 주의를 집중하여, 해당 입력을 System A 또는 B로 라우팅합니다.
- 사회적 학습의 위계: 기본 관찰/연합 학습(포유류, 조류) → 행동 복사(침팬지) → 안내된 학습(부모-자녀 상호작용) → 고수준 사회 학습(언어를 통한 규범·지식의 문화 전달, 인간만 해당)
- 현재 AI에서는 이 과정이 완전히 외부화되어 있어, 데이터 과학자 팀이 신뢰할 만한 출처의 데이터를 직접 수집하거나 post-training 방법을 사용합니다.

**상상을 통한 학습 (Learning from Imagination)**
- System M은 System A/B를 추론 모드로 전환하고, 입력을 센서 대신 **에피소딕 메모리**에서 라우팅하며, 출력을 내부 시뮬레이션으로 보냅니다.
- **휴식 중 메모리 리플레이:** 활동 중 일시 정지 시 최근 방문한 장소의 장소 세포가 재활성화 → 의사결정, 단기 계획에 연결
- **수면 중 메모리 리플레이:** 비-REM 수면 중 최근 경험을 압축된 시간 척도로 재생 → 장기 기억 통합, 일반화
- **장기 계획:** 도구 사용, 지연된 보상, 반사실적 추론 등 사전 시행착오 없이 문제를 해결하는 능력 → 상상력 기반의 유연한 예견

> **핵심 통찰:** System M의 유연성은 단순한 라우터를 넘어, **질적으로 새로운 학습 체제의 활성화자**로 기능합니다. 이는 System M이 고정된 전이 테이블일지라도 충분히 풍부한 구성으로 고차원 학습을 지원할 수 있음을 시사합니다.

# 4. Evo/Devo 프레임워크: 자율 학습의 부트스트래핑

---

![image_5](https://raw.githubusercontent.com/ohilikeit/automd/master/data/outputs/images/why_ai_systems_dont_learn/image_5.png)

System A, B, M이 상호 의존적이라는 것은 **닭-달걀 문제**를 야기합니다: System A는 행동 생성 데이터가 필요하고, System B는 지각 구조에 의존하며, System M은 다른 시스템의 잘 보정된 오차 신호에 의존합니다. 어떻게 시작할 수 있을까요?

### 생물학의 해법: 진화-발달 (Evo/Devo) 이원성

저자들은 생물학의 **진화적 스케일**과 **발달적 스케일**의 구분에서 해답을 찾습니다:

- **발달적 스케일 (내부 루프):** 유기체의 일생 동안, System A와 B가 환경과 상호작용하며 고정된 System M의 제어 하에 파라미터를 업데이트합니다. 의도적으로 단순화된 시작점에서 점차 복잡성이 증가합니다.
    - 신생아의 시력은 제한적이고, 근육의 자유도도 시너지 근육에 의해 제한됩니다.
    - 이러한 **내적 커리큘럼**은 학습을 점진적으로 복잡한 과제로 유도합니다.
- **진화적 스케일 (외부 루프):** 세대를 걸쳐 메타-파라미터 φ (아키텍처, 초기 가중치, System M의 전이 테이블)가 적합도 함수 L을 최적화하도록 업데이트됩니다.

### 수학적 정의: Bilevel Optimization

이 문제는 **이중-수준 최적화(bilevel optimization)** 문제로 정의됩니다:

```
φ_{t+1} = arg min_{φ_t} L(A_0 : A_K, B_0 : B_K)

조건:
  A_0, B_0, M = Init(φ_t)              ← 초기화 절차
  A_{i+1}, B_{i+1} = Update(M, A_i, B_i, Env)  ← 내부 루프 업데이트 규칙
  Env = 상호작용 환경
```

- **외부 루프 (진화):** 에이전트의 생애 주기 전체가 하나의 데이터포인트이며, φ는 수백만 번의 시뮬레이션된 생애 주기를 통해 최적화됩니다.
- **내부 루프 (발달):** 각 생애 주기 내에서 System A, B가 환경과 상호작용하며 학습합니다.

### 핵심 도전과제

- **계산 효율성:** 외부 루프에서 φ를 최적화하려면 수백만 번의 내부 루프(생애 주기)를 시뮬레이션해야 하며, 각 내부 루프는 수백만 데이터포인트에 대한 학습을 포함합니다.
- **확장성:** Bilevel optimization 기법들이 대규모 아키텍처로 확장될 때 심각한 스케일링 문제가 발생합니다.
- **진화적 커리큘럼:** 환경의 다양성과 예측 불가능성을 점진적으로 증가시키는 커리큘럼을 통해 세 컴포넌트의 공진화를 촉진해야 합니다.

# 5. 결론 및 전망

---

### 왜 자율 학습이 중요한가?

- 동물이나 인간처럼 자율적으로 학습하는 시스템은 **원시 경험으로부터 직접 학습**하여 MLOps 파이프라인 전체를 우회할 수 있습니다.
- 제한된 예제에서 일반화하고, 호기심을 통해 새로운 과제를 탐색하며, 물리적·사회적 세계에 대한 **체화된(embodied), 기반된(grounded)** 이해를 발전시킬 수 있습니다.
- 가정용 로봇부터 과학적 발견까지, 불확실하거나 동적인 환경에서의 실세계 배포에 적합합니다.

### 왜 어려운가?

| 도전과제 | 설명 |
| --- | --- |
| **시뮬레이터와 환경** | System A/B/M을 긴밀하게 결합하여 훈련하려면 사실적이면서도 실시간보다 빠른 시뮬레이션 환경이 필요 |
| **평가와 벤치마킹** | 범용 에이전트일수록 태스크별 벤치마크가 진단 가치를 잃음 → **단위 테스트**(개별 시스템)와 **통합 테스트**(결합 평가)를 구분 필요 |
| **이중-수준 최적화 확장** | 복잡한 환경에서의 평생 학습 최적화는 계산적으로 극도로 요구적이며, 커리큘럼 설계에 민감 |
| **윤리적 문제** | **적응성 vs 통제가능성:** 자율 학습 모드에서 의도된 목표와의 정렬이 더 어려워짐. **정렬 해킹:** 생물학적 에이전트가 프록시 보상을 최적화하여 중독 등 부적응 행동을 보이듯, 인공 에이전트도 유사한 취약성 보유. **과도한 신뢰:** 인간처럼 행동하는 시스템에 대한 의인화와 정서적 애착 위험. **도덕적 지위:** 고통이나 공포와 유사한 신호를 처리하는 시스템의 도덕적 위상 문제 |

### 앞으로의 길

AI 분야는 이미 **런타임 적응**과 **추론 시간 컴퓨팅**이라는 관련 주제로 이동하고 있습니다:
- LLM에서의 test-time training, 검증기 기반 선택, 시너지적 적응, 적응적 검색
- Vision-Language 모델과 음성 인식에서의 런타임 적응
- 로보틱스에서의 온라인 적응 학습, 배포 시점 교정

하지만 이러한 적응들은 여전히 전체적으로 고정된 시스템 위의 **미세한 변형**에 불과합니다. 언어와 새로운 스킬을 통째로 학습하는 아이들과 비교하면 갈 길이 멀지만, 저자들은 **System A-B-M 아키텍처**가 이 문제 공간에 대한 통합된 개념화를 제공하며, 실제 자율 학습 시스템 구축을 위한 경로가 될 수 있다고 결론짓습니다.
